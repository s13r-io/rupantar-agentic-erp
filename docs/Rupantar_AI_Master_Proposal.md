## Part 1 — Executive Summary

India has approximately 50 million micro and small businesses — kiranas, restaurants, pharmacies, garages, salons, fashion shops, coaching centres, and more. They collectively handle ₹95 lakh crore in annual retail and services. Yet 80–85% of them use no operational software at all. The ones that do are underserved by legacy desktop tools that were designed for accountants, not shopkeepers.

The opportunity is not incremental. It is a category creation: moving India's unorganised trade from paper and memory to AI-native operations — delivered through WhatsApp, configurable without code, and adopted without requiring any behavioral change from the shop owner.

> **Design principle:** The shopkeeper does not serve the software. The software serves the shopkeeper. Every agent runs autonomously through WhatsApp — the shop owner's single interaction is a voice note, a photo, or a one-word reply.

### Area 01 — Billing, POS & Invoicing

| | |
|---|---|
| **Addressable Shops** | ~50M addressable |
| **Current Penetration** | 15–20% |
| **Existing Market Leaders** | Tally (38%), Vyapar (13%), Marg (11%), Busy (9%), MyBillBook (7%) |

**Market Reality**
Tally's dominance is accountant-driven, not shopkeeper-driven. Vyapar and MyBillBook have traction on mobile but still require a UI interaction for every bill. 80%+ of India's shops remain on paper because no product meets them via voice in their own language.

**⚙ WHAT WE BUILD**
**Voice-first Billing Agent** — shopkeeper speaks an order in Hindi/Hinglish, agent generates a GST-compliant bill, identifies the customer, applies pending discounts or credit, and sends it via WhatsApp. Zero screen interaction for the owner.

**KEY AGENT FEATURES**

- **Voice Billing:** Parses spoken orders (Hindi/Hinglish/regional) into structured bills — handles local aliases ('parle g', 'adha kilo doodh'), unit variations, and combo deals
- **Smart Scan:** Photo or barcode of product → auto-identifies SKU, fetches price, adds to bill — no manual catalogue search
- **Customer Memory:** Recognises returning customers by phone number or UPI ID — auto-applies saved price, outstanding credit, and preferred items
- **GST Engine:** Auto-assigns HSN codes and tax slabs, generates e-invoice for B2B buyers above threshold, validates bill before print
- **Day Close Agent:** End-of-day: tallies cash + UPI + credit sales, flags discrepancies, posts to accounting ledger, sends WhatsApp summary to owner
- **Offline Sync:** Queues all transactions during internet outage, syncs with conflict resolution when connectivity returns

## Part 4 — The Agent Builder Platform

The Agent Builder is the technical infrastructure that makes the entire plan commercially viable. Without it, customising agents for each shop requires engineering — making cost-per-deployment prohibitive. With it, a trained non-developer partner deploys a working multi-agent system for any shop in under 60 minutes.

### Platform Architecture — Four Layers

| Layer | What It Is | Primary User | Analogous To |
|---|---|---|---|
| **L1: Agent Library** | 160+ pre-built, tested, production-ready agent nodes organised by vertical and function. Each is a working agent with standard inputs/outputs, connectors, and default prompts. | Partners: deploy out-of-the-box bundles for standard shops. Shop live in <15 minutes. | Salesforce AppExchange / Zapier templates |
| **L2: Visual Flow Builder** | Drag-and-drop canvas to connect agent nodes, configure triggers (WhatsApp message, UPI credit, timer, stock threshold), and set shop-specific parameters. No code required. | Partners: handle 80% of implementations. 2-day training to proficiency. | n8n canvas / Dify workflow builder — purpose-built for Indian SMB context |
| **L3: Config Studio** | Structured mobile wizard for shop-specific data: product catalog upload, supplier WhatsApp numbers, customer list, language preference, agent tone (formal/Hinglish/regional). Converts raw shop data to agent context. | Partners: fill in on-site during 60-minute shop visit. Mobile-first, works on 4G. | Shopify store setup wizard / Zoho CRM onboarding |
| **L4: Pro Code SDK** | Python/JS SDK to build custom agent nodes, connect non-standard APIs, and publish templates to the community marketplace. Earns higher partner margins and 'Master Builder' certification. | Tier 3–4 technical partners and ex-developers building vertical specialisations. | n8n community nodes / Dify plugin system |

### Core Platform Capabilities

| Capability | Description |
|---|---|
| **WhatsApp-Native Execution** | All agent actions execute through WhatsApp Business API. No separate app required by shop owner or customer. Supports text, voice notes, images, buttons, and payment links. |
| **Vernacular NLP Engine** | Trained on Indian commercial vocabulary across 12 languages — understands 'adha kilo', 'khokha', 'peti', 'tola', local product aliases, and regional business idioms that global LLMs miss. |
| **Offline-First Queue** | All agent actions queue locally during internet outage. Sync with conflict resolution on reconnect. Critical for Tier 2–4 cities with patchy connectivity. |
| **Planner → Executor → Verifier Loop** | Each agent: plans the action ('send credit reminder to Ram'), executes it (sends WhatsApp), verifies outcome (checks if payment received in 48 hrs, retries with escalated tone if not). |
| **Confidence Threshold** | Every agent has a configurable confidence threshold below which it asks owner for confirmation before acting. High-stakes actions (orders >₹500, GST filing) always require explicit approval. |
| **Observability (Langfuse/OpenTelemetry)** | Complete audit trail of every agent decision, input, and action. Partners and Rupantar can replay any agent run. Full trace available within 48 hrs for dispute resolution. |
| **Persistent Shop Memory** | Each shop has an isolated memory store (customer profiles, product catalog, supplier list, transaction history, seasonal patterns). Memory compounds — agents become more accurate with every month of usage. |
| **Multi-Agent Orchestration** | Orchestrator agent classifies owner intent, routes to specialist domain agents, and composes responses across multiple agent outputs. Handles ambiguous or multi-step requests without breaking flow. |

> **Competitive differentiation:** n8n, Dify, and Langflow are excellent horizontal platforms. None of them know what 'khokha' means, have WhatsApp Business API baked in, understand Indian GST flows, or are designed for implementation by a pharma MR or Tally reseller with 2 days of training. We are not competing with them — we are building on the same principles for a radically different user and use case.

| Moat Type | How It's Built | Why Incumbents Can't Replicate |

| **Data Flywheel** | Every transaction, voice note, and WhatsApp exchange trains the system — local product names, regional price norms, supplier networks, customer behaviour.
